# Промпт для генерации Telegram finance bot

Скопируй этот промпт в Cursor, ChatGPT, Claude или другой инструмент для vibe coding.

---

Нужен Telegram-бот для учета личных финансов.

## Цель
Сделай простого, аккуратного и рабочего Telegram-бота, который помогает фиксировать доходы и расходы в личном бюджете.

## Техстек
- используй язык программирования: `python@3.13.12`
- данные храни локально в рантайме
- для переменных окружения используй `.env` файл
- для Telegram ключа используй переменную: `TG_BOT_API_KEY`
- архитектура должна быть простой, чистой и понятной новичку
- код должен быть разбит по файлам, а не свален в одну простыню, потому что страдать и так есть от чего

## Базовый функционал
Сделай бота, который умеет:

1. Принимать быстрые записи текстом:
   - `-350 кофе`
   - `+50000 зарплата`
   - `-1290 продукты - пятёрочка`
   - `+5000 фриланс | лендинг`

2. Распознавать:
   - `+` как доход
   - `-` как расход
   - сумму
   - категорию
   - необязательную заметку

3. Хранить операции локально в файле JSON.

4. Поддерживать команды:
   - `/start`
   - `/help`
   - `/today` — сводка за сегодня
   - `/month` — сводка за текущий месяц
   - `/balance` — общий баланс
   - `/categories` — расходы по категориям за текущий месяц
   - `/last` — последние 10 операций
   - `/undo` — удалить последнюю операцию

5. Делать понятные ответы пользователю на русском языке.

## UX-требования
- бот должен быть дружелюбным, кратким и понятным
- ответы не должны быть перегружены
- текст должен быть аккуратно оформлен
- ошибки ввода должны объясняться человеческим языком
- если формат не распознан, бот должен показать 1-2 примера корректного ввода

## Технические требования
- используй асинхронную библиотеку для Telegram-бота
- предпочти `python-telegram-bot`
- зависимости вынеси в `requirements.txt`
- добавь `.env.example`
- добавь `README.md` с пошаговым запуском
- добавь `.gitignore`
- сделай структуру проекта примерно такой:

```text
finance_bot/
  app/
    __init__.py
    bot.py
    storage.py
    utils.py
  data/
  .env.example
  .gitignore
  README.md
  requirements.txt
  main.py
```

## Логика хранения
- для каждого пользователя храни операции отдельно
- каждая операция должна содержать:
  - `id`
  - `user_id`
  - `amount`
  - `category`
  - `note`
  - `created_at`
- если категории нет, ставь `прочее`

## Критерии качества
- код должен запускаться без доработок после установки зависимостей и заполнения `.env`
- без базы данных
- без Docker
- без вебхуков
- запуск через long polling
- без лишнего оверинжиниринга
- код должен быть читаемым, с понятными именами функций
